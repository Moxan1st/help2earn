Help2Earn
Help2Earn是一个基于 SpoonOS 的 DePIN（去中心化物理基建）网络。我们利用 Gemini 的多模态能力作为验证节点，激励全球用户通过简单的『拍照赚钱』模式，构建世界上最精准的无障碍数据层。

核心场景 (Scenarios)
两个故事，分别对应“供应端（赚钱的人）”和“需求端（用数据的人）”。
场景 A：数据矿工 (The Miner)
人物： Kenji，一名深圳南山的大学生。
动作： Kenji 在去学校的路上，发现一家新开的咖啡馆门口有完美的轮椅坡道。他拿出手机，打开 Help2Earn，拍了一张门口的照片并上传。
结果： 3秒后，AI 验证通过，Kenji 的钱包收到了 50 个 tokens 代币（够喝个奶茶）。
价值： 零工经济，人人都能参与。

场景 B：数据消费者 (The User)
人物： Sato 先生，轮椅使用者。
动作： Sato 想去这家咖啡馆，但在 Google Maps 上只看到模糊的标签。他打开 Help2Earn。
结果： 他不仅看到了“有坡道”的标记，还看到了 AI 对坡道的分析：“坡度小于 5 度，路面平整，但注意：自动门开启时间较短。”
价值： 真正的出行安全感。

工作流程 (Workflow) - 技术架构图

Input Layer (输入层):
用户前端 (Web/Mobile) 上传图片 + GPS 坐标。
Prompt: "Check for ramp."
Processing Layer (SpoonOS Agent 层):
大模型: 接收图片。
Vision Analysis: 识别图片内容（是坡道还是楼梯？）。
Logic Check: 如果图片是“一只猫” -> 拒绝；如果是“坡道” -> 提取元数据（类型）。

Settlement Layer (结算层 - 模拟):
AI 返回结果 {valid: true}。
Agent 触发智能合约（Smart Contract）。
合约账户向用户地址转账 Token。
Storage Layer (数据层):
将 AI 分析出的结构化数据（如：“海岸城星巴克，坡道，状况良好（无维修），时间戳”）存入数据库，供前端地图调用。

功能：看+帮
看：UI背景显示用户当前半径 200 m 的地图，标出所有有坡道（坡道小图标）、无障碍厕所（无障碍厕所小图标）、升降电梯（直梯小图标）、轮椅借用处（轮椅借用处小图标）和无障碍友好的商场和医院(小图标密集后变成大图标)。点击大图标分散成小图标，点击小图标查看详情图片。
帮：
1.地图背景中下方悬空一个拍照按钮，按下这个按钮读取用户当前位置 GPS 坐标＋上传照片。
2.图片由 SpoonOS 交给多模态大模型，请求类似于：“确认是否为无障碍设施”。
3.Agent判断1大模型返回是或者否；如果返回是，则进入
判断2当前位置是否有同类型标记的无障碍设施，判断为是则检查上一张照片的时间戳进入判断3是否是15天内的，
如果是 15天内 的 -> 拒绝（防刷单）。
如果是 15天前 的 -> 允许覆盖/更新（视为“维护数据”），给用户发一半的奖励（比如 25 Token）。
如果判断1为否，将该无障碍设施添加到数据库中，部分数据上链，上链的信息下面会描述，并且 Agent 触发智能合约向用户钱包地址转账Token。

上链的东西 (On-Chain)： 只存一个 Hash 值（哈希指纹） 和 交易记录。
例如：链上只记录 User A 在 Time T 验证了 Location X，获得 10 Tokens。这保证了激励机制不可篡改
